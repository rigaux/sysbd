  • Quelles informations sur l’image avant et l’image après sont exactes?

    1. L’image après ne contient que des nuplets validés par un commit

    2. L’image avant ne contient que des nuplets validés par un commit

    3. Toutes les lectures se font dans l’image avant

    4. Toutes les lectures se font dans l’image après

  • Dans quel mode a-t-on besoin de conserver plus d’une version dans l’image avant?

  1. En repeatable read

  2. En read committed

  3. En serializable

  • Jusqu’à quand doit-on garder une version \(e_i\) estampillée à l’instant \(i\)

  1. Pour toujours

  2. Tant que la transaction qui a écrit \(e_i\) est active

  3. Tant qu’il existe une transaction qui a débuté avant \(i\)

  4. Tant qu’il existe une transaction qui a déjà lu \(e_i\)
